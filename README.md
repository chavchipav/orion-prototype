# ЯндексАгро — кликабельный прод-макет

Единый кабинет уровня реального коммерческого продукта, смоделированный на эталоне **АгроМон** (российская FMS,
Bayer). Для показа Genesis/Надежде и защиты Growth Lab (мандат Хаустова — кликабельный прототип). Статус и
бэклог — [`../ПРОГРЕСС_прототипа.md`](../ПРОГРЕСС_прототипа.md); ТЗ-сценарии — [`../Сценарии_продакшн_v2.md`](../Сценарии_продакшн_v2.md).

## Запуск
```bash
cd agrotech/prototype/app
npm install      # если ещё не ставили
npm run dev      # → http://localhost:5173
```
Сборка статики для показа без dev-сервера: `npm run build` → `dist/`.
Сборка в **один самодостаточный HTML** (двойной клик, без установки): `npm run build:single` → `dist-single/index.html`.

## Архитектура
Единая тёмная GIS-оболочка `UnifiedShell.tsx` рендерит **все роли** в одном стиле: сайдбар с пер-ролевой
навигацией (`ROLE_NAV`), переключатель ролей в шапке, роутинг контента по ключу экрана (`ScreenKey`). Карты —
реальные тайлы **Leaflet + спутник Esri World Imagery** (без API-ключа) с **реальными контурами полей из OSM**,
точно лежащими на снимке. Единый язык карт (зум-контролы, легенда, слои, fit) — `components/MapKit.tsx`. Акцент —
красный Яндекса (`brand` `#fc3f1d`).

## Роли и разделы (переключатель «Кабинет» в шапке)
- **Агроном / Хозяйство (18 разделов, паритет навбара АгроМон + Планировщик и Опыты):** Клиенты · **Карта** (спутник, слои Снимок/NDVI/
  Контрастный/Культуры, **реальные контуры полей OSM на снимке**, зум-контролы, GPS-треки по контуру, Хронология) ·
  Вегетация · Осмотры · Рекомендации · Техкарты (+ СЗР) · Питание · Кадастр (Росреестр-оверлей 1:1 с полем) ·
  Работы · **Планировщик** (Агрокопайлот: проактивные работы по фенофазе+погоде, окно день/ночь) · Календарь ·
  Севооборот · Погода · Справочники · Маркетплейс (5 категорий) · Аналитика · **Опыты** (двухфакторные эксперименты
  → разложение дисперсии → решение) · Настройки.
- **Семеновод (8 разделов, «Genesis OS» — кабинет Надежды):** Сводка · Демо-сеть «Баттл» · **Клиентские поля**
  (drill-in: обзор пинами клиентов → клик → реальное поле клиента на снимке его региона) · Гибриды · Zone-fit ·
  Каталог · предложение · Контракты/партии · Воронка · Производство.
- **Владелец** (read-only портфель, общая карта с агрономом) · **Дистрибьютор** · **Банк/страховщик**.

## Стек
Vite + React 19 + TypeScript · Tailwind v4 (`@tailwindcss/vite`) · Recharts (графики) · Leaflet/react-leaflet
(карты) · lucide-react (иконки). Все данные — мок.

## Структура `src/`
- `UnifiedShell.tsx` — единая оболочка: `ROLE_NAV`, роутер `Screen`, переключатель ролей.
- `store.tsx` — роль/экран/навигация (`ScreenKey`, `useApp`). `seedStore.tsx` — мутабельный домен семеновода.
- `agronom/` — экраны кабинета агронома (MapWorkspace, Scouting, Vegetation, Techcards, Cadastre, Works,
  Weather, Analytics, Rotation, Recommendations, Reference, Clients, Nutrition, Calendar, Marketplace, Settings,
  Chronology + селекторы маркетплейса).
- `agronomData.ts` / `agronomData2.ts` — мок-домен агронома (NDVI/осмотры/техкарты/GPS/погода/кадастр/…);
  **`agronomFieldsGeo.ts`** — реальная геометрия полей (контуры OSM, генерится `scripts/fetchFields.mjs`).
- `screens/seed/` — 8 экранов семеновода + `SeedMap.tsx` (drill-in Leaflet). `seedData.ts` — мок-домен;
  **`seedFieldsGeo.ts`** — реальные контуры полей клиентов (генерится `scripts/fetchClientFields.mjs`).
- `components/MapKit.tsx` — общие контролы карт (зум/легенда/слои/fit). `screens/Distributor.tsx`,
  `screens/Fintech.tsx` — кабинеты дистрибьютора и банка. `data.ts` — роли/тарифы; `ui.tsx` — примитивы.
- `scripts/` — build-скрипты геоданных (OSM/Overpass → закоммиченные `*Geo.ts`, рантайм офлайн).
