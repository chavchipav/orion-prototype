# ЯндексАгро — прототип: запуск и тест

Кликабельный макет единого кабинета (агроном · владелец · семеновод · дистрибьютор · банк). Прокликайте
сценарии, и при желании меняйте/дорабатывайте в своём **Claude Code** — проект к этому готов (см. `CLAUDE.md`).

---

## 0. Просто посмотреть — без установки (файл `.html`)
Если нужно только **потыкать прототип**, попросите у Саши файл **`ЯндексАгро.html`** — это весь прототип в одном
файле. Скачайте и **откройте двойным кликом** (лучше в Chrome) — установка не нужна. Нужен только интернет (карты
подгружают спутниковые снимки). Этот вариант — для просмотра; чтобы **менять код**, используйте установку ниже.

> Сгенерировать такой файл самим из исходников: `npm run build:single` → `dist-single/index.html`.

---

## 1. Что нужно (один раз)
- **Node.js 18+** — поставить с https://nodejs.org (кнопка LTS). Проверка в терминале: `node -v`.
- **Браузер Chrome** (рекомендуется) и **интернет** — карты подгружают спутниковые снимки.
- *(опционально, для правок)* **Claude Code** — https://claude.com/claude-code.

## 2. Запуск (3 шага)
Распакуйте архив, откройте папку в терминале и выполните:
```bash
npm install      # установка зависимостей (1–2 мин, один раз)
npm run dev      # запуск
```
Откройте в браузере адрес, который покажет терминал — обычно **http://localhost:5173**.
Остановить сервер — `Ctrl+C` в терминале.

> Если порт занят — Vite сам предложит другой адрес (или запустите `npm run dev -- --port 5174`).
> Если карта пустая/серая — подождите пару секунд и проверьте интернет (тайлы грузятся из сети).

## 3. Как пользоваться
Вверху справа — переключатель **«Кабинет: …»**: меняет роль (Агроном, Владелец, Семеновод, Дистрибьютор,
Банк). Слева — меню разделов роли. Состояние сбрасывается обновлением страницы (F5).

## 4. Что прокликать (основные сценарии)

**Агроном — «память о земле» и единый цикл**
1. **Карта** → клик по полю (где статус-флаг «требует решения») → кнопка **«Сезон поля»**: лента фенофаз, кривая
   NDVI, жизненный цикл проблемы, план/факт, **ROI защиты**. Переключите год — кокпит за прошлый сезон.
2. **Рекомендации** → на открытой проблеме **«Принять и в работу»** → загляните в **Осмотры** и **Работы**: та же
   проблема уже в новом статусе (одна живая система, не три таблицы).
3. **Карта**: вкладки **Поля / Спутниковые снимки / GPS мониторинг**, слои внизу (Снимок / NDVI / Контрастный /
   Культуры), кнопки зума. Поля — **реальные контуры по снимку**. **Кадастр** — оверлей Росреестра 1:1 с полем.

**Маркетплейс — подбор по полевым испытаниям (двусторонние петли)**
4. **Агроном → Маркетплейс → Семена**: пройдите визард (культура → тип → … → регион) → результаты с «проверено
   полем» и отзывами фермеров → откройте паспорт гибрида → **«Оставить заявку семеноводу»**.
5. Переключите роль на **Семеновод → Каталог · предложение → вкладка «Заявки»**: заявка пришла → **«Завести
   демо»** → она появляется в **Демо-сети «Баттл»**. Петля замкнулась.
6. **Агроном → Маркетплейс → СЗР / Удобрения / Техника / Софт**: тот же движок подбора. **«Оставить заявку»** →
   переключите роль на **Дистрибьютор**: заявка уже в его борде (взять → предложение → сделка) + нотификация.

**Семеновод — drill-in карта клиентов**
7. **Семеновод → Клиентские поля**: национальный обзор — пины клиентов (цвет = гибрид). **Клик по клиенту** →
   камера летит к его **реальному полю на снимке** региона. Кнопка **«← Все поля клиентов»** — назад.

**Владелец** — та же карта, что у агронома, но read-only портфель (без редактирования).

> Полный 5–7-минутный нарратив-прогон — у Саши (CPO), файл `Демо-сценарий_сквозной.md` в репозитории.

## 5. Правки в своём Claude Code
Откройте папку проекта в Claude Code и просто опишите задачу — он прочитает `CLAUDE.md` и сориентируется.
Примеры запросов:
- «Поменяй акцентный цвет на синий» · «Добавь раздел N в кабинет агронома» ·
- «Смени локацию полей фермы на Краснодарский край» (он отредактирует `scripts/fetchFields.mjs` и перегенерит) ·
- «Добавь новую категорию в маркетплейс».
После правок просите его прогнать `npm run build` (проверка) и показать результат в браузере.

## 6. Полезное
- Сборка статики (без dev-сервера): `npm run build` → папка `dist/`.
- Перегенерировать реальные контуры полей: `node scripts/fetchFields.mjs` (ферма) /
  `node scripts/fetchClientFields.mjs` (клиенты семеновода) — нужен интернет.
- Технический гид по коду — `CLAUDE.md`; обзор архитектуры — `README.md`.

## 7. Если что-то не работает
- `npm install` падает → проверьте `node -v` (нужен 18+), интернет, попробуйте удалить `node_modules` и
  `package-lock.json` и снова `npm install`.
- Белый экран → откройте консоль браузера (F12) и пришлите ошибку Саше.
- Карты не грузятся → это сеть/корп-фильтр на тайлы Esri; попробуйте другую сеть.
