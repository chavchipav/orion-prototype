# ИИ-ассистент Орион — безопасное подключение ChatGPT (бэкенд-прокси)

Чтобы ассистент работал у **всех** посетителей и при этом ключ OpenAI **не светился** в браузере,
ключ держим на сервере-прокси (Cloudflare Worker, бесплатно). В браузер уходит только URL прокси — это не секрет.

## Шаг 1. Развернуть прокси (Cloudflare Worker, ~3 минуты)
1. Зайдите на https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Create Worker**.
2. Назовите, например, `orion-ai`, нажмите **Deploy** (создастся пустой воркер).
3. **Edit code** → вставьте целиком содержимое `worker.js` (рядом) → **Deploy**.
4. Откройте воркер → **Settings → Variables and Secrets → Add** →
   - тип **Secret**, имя `OPENAI_API_KEY`, значение — ваш ключ `sk-...` → **Save and deploy**.
5. (Опционально) в `worker.js` в `ALLOW_ORIGINS` укажите ваш домен (по умолчанию `https://atsesl1.github.io`).
6. Скопируйте URL воркера, вид: `https://orion-ai.<ваш-аккаунт>.workers.dev`.

> Альтернатива — Vercel/Netlify function: логика та же (forward в `api.openai.com` с ключом из env).

## Шаг 2. Подключить прокси к кабинету
В файле **`app.html`** добавьте ОДНУ строку сразу после `<head>` (URL — ваш из шага 1):

```html
<script>window.ORION_AI_PROXY = "https://orion-ai.ВАШ-АККАУНТ.workers.dev";</script>
```

Готово. Перезалейте `app.html` — ассистент заработает у всех **без ввода ключа**, ключ остаётся на сервере.

## Как это работает
- Есть `window.ORION_AI_PROXY` → ассистент ходит в прокси (ключ на сервере). ✔️ безопасно, работает у всех.
- Нет прокси, но пользователь ввёл свой ключ (иконка ключа в ассистенте) → ходит напрямую в OpenAI своим ключом.
- Нет ни того, ни другого → ассистент отвечает демо-заглушками.

## Безопасность / контроль расходов
- Ключ **никогда** не попадает в `app.html` (проверено: в бандле 0 ключей). В публичный файл секрет не вшиваем.
- Прокси ограничивает модели (`gpt-4o-mini`/`gpt-4o`) и `max_tokens` — защита от дорогих запросов.
- Рекомендуется: в OpenAI выставить **месячный лимит трат (Usage limits)**; в Cloudflare включить **Rate limiting** на воркер.
- ⚠️ Прежний ключ `sk-proj-Gvfrj6…` скомпрометирован ранее — создайте **новый** ключ для прокси, старый отзовите.
